  - 自定义负载因子和初始容量：
    - 若业务中 “查询多、插入少”，可适当降低负载因子（如 0.6），减少冲突；
    - 若 “内存紧张、插入多”，可适当提高负载因子（如 0.85），提升空间利用率；
  - 避免大对象作为 key：大对象的`hashCode()`计算耗时，且`equals()`对比效率低，建议用对象的主键（如 Long 型 id）作为 key；
  - 批量操作优化：批量插入数据时，先初始化足够大的`HashMap`，避免扩容；批量查询时，用`forEach`替代多次`get()`。
